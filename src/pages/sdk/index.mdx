# SDK

<span className="lead">
  The Adamite SDK allows you to connect your client to your Adamite instance.
</span>

## Install the SDK

<section>

<div>

To get started, install the SDK using `yarn` or `npm`.

</div>

```bash
# Using NPM
npm install --save @adamite/sdk

# Using Yarn
yarn add @adamite/sdk
```

</section>

---

## Initialize the SDK

<section>

<div>

Once installed, you can initialize the SDK to connect it to your Adamite instance.

<br />

<Callout intent={Intent.PRIMARY} icon={null}>
  On first run, Adamite will generate an SDK key for you to use. The SDK key will be printed
  to the console when you run your server.
</Callout>

</div>

```jsx
import adamite, {
  AuthPlugin,
  DatabasePlugin,
  FunctionsPlugin,
} from "@adamite/sdk";

adamite()
  .use(AuthPlugin)
  .use(DatabasePlugin)
  .use(FunctionsPlugin)
  .initializeApp({
    url: "http://localhost:9000",
    apiKey: "your_api_key",
  });
```

</section>

## Understanding the SDK

<section>

<div>

Adamite's SDK strives to be easy to understand by following some consistent patterns.

</div>

</section>

### Get an app instance

<section>

<div>

When you call the `adamite` function, it returns an App. An App is the root of the Adamite SDK.

</div>

```js
const app = adamite();
```

</section>

<section>

<div>

You can optionally pass an app name to the `adamite` function if you want to use more than one app in your client. If you don't pass this string, the app name `default` is used.

</div>

```js
const secondaryApp = adamite("secondary");
```

</section>
<br />

### Access plugins

<section>

<div>

Once you have an App instance, you can access the core plugins with convenience methods.

</div>

```js
const database = adamite().database();
const functions = adamite().functions();
const auth = adamite().auth();
```

</section>

<br /><br />

<section>

<div>

You can access custom plugins using the `plugin` method.

</div>

```js
const customPlugin = adamite().plugin("customPlugin");
```

</section>
